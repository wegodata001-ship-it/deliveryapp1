## היסטוריית דיווח (תמצית)

1. ניווט תשלומים — דווח חוזר; תיקונים נקודתיים לא הספיקו → **הוחלט Rebuild**.
2. חוב פתוח / מונה תשלומים — דווח לאחרונה.
3. איפוס יתרה — דווח (עדכון מיידי ל-DB) → שונה ל-preview; דורש אימות מלא ב-Rebuild.
4. איפוס עמלה / Y−X — מפרט עסקי מפורט; מיושם חלקית.
5. UX רוחב/גלילה/קומפקטיות — דרישות סותרות לאורך זמן → ייפתר ב-UI חדש.
